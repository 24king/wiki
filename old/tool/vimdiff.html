    1. 启动vimdiff的方式
      1. vimdiff FILE_LEFT FILE_RIGHT
      2. vimdiff FILE_LEFT --> :vertical diffsplit FILE_RIGHT
    2. 光标移动
      1. :set noscrollbind
      2. ]c 差异间快速移动
      3. [c 反向跳转
    3. 文件合并
      1. dp (diff "put") 讲当前文件中的差一点复制到另一边
      2. do (diff "get")
    4. 手动刷新
      1. diffupdate
    5. 上下文的展开和查看
      1. set diffopt=context:3
      2. zc, zo

    当使用svn diff 命令比较某个文件的修改前后时, svn diff -r4420 ng.c 实际会向默认的diff程序发送如下命令
    -u -L ngx_http_limit_req_module.c (revision 4420) -L ngx_http_limit_req_module.c (working copy) .svn/tmp/tempfile.tmp ngx_http_limit_req_module.c
    传递的参数比较多,所以可以写一个脚本来实现参数的处理功能;
    #!/bin/sh

    1. 去掉前5个参数

    shift 5

    1. 使用vimdiff比较

    vimdiff "$@"

    svn还提供了在配置文件中修改diff默认程序的功能，这样就不用每次使用svn diff时再指定--diff-cmd了。
    diff-cmd = /usr/local/bin/diffwrap.sh
    以后使用SVN就没有什么问题了再
